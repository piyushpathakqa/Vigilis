Vigilis

The AI agent that writes your Playwright tests — and fixes the ones your UI breaks.

Point it at your app. It generates the specs, runs them as a CI gate, and self-heals the locators that break from harmless UI changes — without ever hiding a real bug. Every fix ships with a signed receipt you can verify.

One loop, five behaviours

01

Generate

Point it at a URL → it explores the app and writes a runnable Playwright spec.

02

Gate

The spec runs in CI as a required check — failing tests block the deploy.

03

Triage

On failure it classifies the cause: real bug vs DOM drift vs flake.

04

Heal

For drift it rewrites the locator, verifies green, and opens a PR — never masking real bugs.

05

Verify

Every run is sealed into a signed, tamper-evident receipt — independently verifiable, not “trust me”.

Self-healing QA you can audit.

Most self-healers ask you to trust a dashboard. Vigilis seals every tool call and model decision into a hash-chained, signed receipt — notarised independently by Treeship. You can verify exactly what the agent did, in what order, unaltered.

Never a false green

It won’t heal a real bug.

Triage distinguishes harmless DOM drift (safe to heal) from genuine behaviour change (must fail). When it’s unsure, it fails loudly — and the decision plus its rationale go into the receipt, so the call is auditable.
